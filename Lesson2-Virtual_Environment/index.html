It's time to set up a Virtual Environment. A virtual environment is a tool that helps to keep dependencies required by different projects separate by creating isolated python virtual environments for them. This is one of the most important tools that most of the Python developers use.

$ sudo apt install -y python3-venv # Now you are installing the venv module

$ cd ~/work/project_01 # you can use ~ instead of $HOME

$ python3 -m venv my_env # create an environment (my_env is the special directory in the /home/lyaki4/work/project_01/ with a lot of directories and files inside)

$ ls my_env # what inside of directory my_env
bin  include  lib  lib64  pyvenv.cfg
$ $ ls my_env/bin/ # what inside of bin in derectory my_env
Activate.ps1  activate  activate.csh  activate.fish  pip  pip3  pip3.10  python  python3  python3.10

Great! You have created your first "sandbox". Next step is activation.

$ source my_env/bin/activate
(my_env) lyaki4@ubuntu:~/work/project_01$

(my_env) before command prompt indicate that you are into the virtual environment now.

$ nano hello.py # open the simple text editor "nano" and create file "hello.py"

Type out print("Hello, World!"), then press CTRL+X, Y, and ENTER

So, it's time to execute your first programm.

(my_env) lyaki4@ubuntu:~/work/project_01$ python hello.py
Hello, World!

So, you can work in python command line or with file scripts. Moreover, you can use special programms for writing code (IDE - Integrated Development Environment).

Finaly, to leave the environment, type the command deactivate and you will return to your original directory.

(my_env) lyaki4@ubuntu:~/work/project_01$ deactivate
lyaki4@ubuntu:~/work/project_01$
Since today you know following:
ls - list directory contents (you can use that command with keys -alF, or activate the aliases in your profile)
source - Execute commands from a file in the current shell
nano - nano is a small and friendly editor
